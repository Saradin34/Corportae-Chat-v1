# Corporate Chat — Desktop (Electron)

Нативное десктоп-приложение, которое оборачивает веб-интерфейс Corporate Chat.
Один и тот же билд работает с любым сервером (локальный Docker или сервер
компании) — при первом запуске спрашивается адрес сервера.

## Возможности
- Установщик для Windows (`.exe`, NSIS), macOS (`.dmg`), Linux (`AppImage`, `.deb`)
- Ярлык на рабочем столе и в меню «Пуск»
- **Сворачивание в панель задач**: кнопка [X] не закрывает приложение, а
  сворачивает окно в панель задач (где «Пуск»). Приложение продолжает работать,
  его кнопка остаётся в панели задач. Полностью закрыть можно так:
  ПКМ по кнопке в панели задач → «Закрыть окно», либо меню «Файл → Закрыть
  приложение» (Ctrl+Q).
- Один экземпляр: повторный запуск разворачивает уже открытое окно
- Меню: смена сервера, масштаб, полный экран, перезагрузка, сворачивание
- Внешние ссылки открываются в системном браузере

## Требования для сборки
- Node.js 18+ и npm
- Иконки уже готовы в `assets/`: `icon.ico` (Windows, многоразмерная),
  `icon.png` 1024×1024 (macOS/Linux). Менять ничего не нужно.

## Установка зависимостей
```bash
cd desktop
npm install
```

## Запуск в режиме разработки
```bash
npm start
```

## Сборка под Windows

Есть три варианта (по убыванию требований к правам):

```bash
npm run dist:win            # Установщик .exe (NSIS) — нужен Developer Mode/админ
npm run dist:win:portable   # Один портативный .exe — БЕЗ прав администратора
npm run dist:win:dir        # Распакованная папка win-unpacked — самый надёжный
```

macOS (.dmg): `npm run dist:mac` · Linux (AppImage+deb): `npm run dist:linux`

Результат — в папке `desktop/dist/`.

> Проще всего запустить `python setup.py` → пункт 3 — он спросит тип сборки и
> при ошибке сам переключится на вариант без прав администратора.

## ⚠️ Сборка без прав администратора (ошибка "Cannot create symbolic link")

Если при `npm run dist:win` появляется:
```
ERROR: Cannot create symbolic link : клиент не обладает требуемыми правами
... winCodeSign ... libcrypto.dylib
```
Это **не ошибка приложения**. electron-builder скачивает набор для подписи кода
(`winCodeSign`), внутри которого есть macOS-симлинки. Windows не создаёт симлинки
без **Режима разработчика** или прав администратора. Решения (любое на выбор):

1. **Портативная сборка (рекомендуется, без прав):**
   ```bash
   npm run dist:win:portable
   ```
   Получите один `CorporateChat-portable-2.0.0.exe` — запускается без установки.

2. **Сборка в папку (без прав):**
   ```bash
   npm run dist:win:dir
   ```
   Папка `dist/win-unpacked/` — внутри `Corporate Chat.exe`. Можно скопировать
   её на любой ПК и просто запускать (или раздать через сетевую папку/GPO).

3. **Включить Режим разработчика Windows** (часто доступно без админа):
   Параметры → Конфиденциальность и безопасность → Для разработчиков →
   включить «Режим разработчика». После этого работает обычный `npm run dist:win`.

4. **Запустить терминал от имени администратора** и повторить `npm run dist:win`.

> Во всех вариантах автоподпись отключена переменной
> `CSC_IDENTITY_AUTO_DISCOVERY=false` (см. `setup.py`) — это и убирает обращение
> к winCodeSign. Иконка приложения берётся из `assets/icon.ico`.

> Примечание: собирать под конкретную ОС лучше на этой же ОС (Windows .exe — на
> Windows и т.д.). Для кросс-сборки используйте CI или образ `electronuserland/builder`.

## Как это работает
- `main.js` — главный процесс: создаёт окно, хранит адрес сервера в
  `userData/config.json`, показывает экран настройки при первом запуске.
- `preload.js` — безопасный мост (`contextIsolation`) для экрана настройки.
- Приложение просто загружает веб-интерфейс с указанного сервера, поэтому
  все функции (чаты, файлы, AD) работают как в браузере.
- Кнопка [X] сворачивает окно в панель задач (а не закрывает).

## Первый запуск пользователя
1. Установить `.exe` (двойной клик, мастер установки).
2. Запустить «Corporate Chat» с рабочего стола.
3. Ввести адрес сервера компании (например `http://chat.company.local`).
4. Войти (локально или через Active Directory).

## Поведение окна
- Кнопка [X] — сворачивает окно в панель задач (если включено «Оставлять
  приложение в трее после закрытия» в Настройки → Основное; по умолчанию вкл).
- Полное закрытие: ПКМ по кнопке приложения в панели задач → «Закрыть окно»,
  либо меню «Файл → Закрыть приложение» (Ctrl+Q).
- Повторный запуск ярлыка разворачивает уже открытое окно.

## Автозапуск
- В Настройки → Основное → «Автоматический запуск при старте» включает запуск
  при входе в Windows/macOS (через `app.setLoginItemSettings`). Приложение
  стартует свёрнутым (`--hidden`), чтобы не мешать.

## Прокси
- Настройки → Подключение → «Не использовать прокси» переключает сессию
  Electron между `direct` (без прокси) и `system` (системные настройки).

## Обновление функционала в desktop

Desktop-приложение загружает веб-интерфейс с сервера, поэтому новый функционал
появляется после обновления backend/nginx на сервере. Чтобы Electron не держал
старые JS/CSS:

- при запуске приложение очищает web cache/service-worker cache;
- меню **Файл → Жёстко обновить (очистить кэш)** делает `reloadIgnoringCache`;
- в веб-настройках desktop доступны кнопки **Обновить веб-интерфейс** и
  **Сменить сервер**;
- `Ctrl+R` — обычная перезагрузка, `Ctrl+Shift+R` — жёсткая перезагрузка.

Если после деплоя серверной версии desktop всё ещё показывает старый интерфейс:
1. Файл → Жёстко обновить (очистить кэш)
2. Полностью закрыть приложение через Файл → Закрыть приложение
3. Запустить снова

## Настройка сервера

Адрес сервера хранится в:

- Windows: `%APPDATA%/Corporate Chat/config.json` (путь Electron `userData`)
- macOS/Linux: соответствующая папка `userData`

Поменять адрес можно через:

- Файл → Сменить сервер…
- Настройки внутри приложения → Основное → Сменить сервер
