# Новые возможности Corporate Chat (v2.1 / v2.2 / v2.3)

## v2.3 — Прокачка админ-панели + масштабирование

### 📊 Дашборд с графиками
- Новая вкладка **«Дашборд»** (по умолчанию): встроенные SVG-столбчатые графики
  «Сообщения за 7 дней» и «Новые пользователи за 7 дней» (без внешних библиотек,
  работают и в песочнице превью). Доп. плитки: каналы, SSO-учётки, админы, чаты.
- `GET /api/admin/stats` теперь отдаёт `messages_per_day`, `users_per_day`,
  `channels`, `sso_users`.

### 👥 Управление пользователями
- **Фильтры**: по роли (админ/юзер), источнику (local/AD/SSO), статусу (активен/заблокирован).
- **Массовые действия** (чекбоксы): заблокировать/разблокировать, назначить/снять
  админа, удалить — `POST /api/admin/users/bulk`.
- **Экспорт в CSV** (с учётом фильтров) — `GET /api/admin/users/export`.
- **«Войти как» (impersonate)** — вход под пользователем для разбора проблем,
  с записью в аудит — `POST /api/admin/users/{id}/impersonate`.

### 📢 Управление каналами
- Новая вкладка **«Каналы»**: создать канал, подписать всех/выбранных,
  назначать **авторов** (кто может публиковать).
- Эндпоинты: `GET/POST /api/admin/channels`,
  `POST /api/admin/channels/{id}/subscribers`,
  `POST /api/admin/channels/{id}/authors/{user_id}`.

### 📋 Журнал аудита
- **Фильтры** по администратору/деталям и по типу действия + **экспорт в CSV**.
- Логируются новые действия: массовые операции, impersonate, каналы, изменение
  настроек, экспорт, авто-/ручная очистка.
- `GET /api/admin/audit?q=&action=`, `/api/admin/audit/actions`, `/api/admin/audit/export`.

### 🗑️ Политики хранения (retention, комплаенс)
- В «Настройках»: «Удалять сообщения старше N дней» (0 = вечно) + опция удаления
  файлов-вложений. Фоновая задача чистит ежедневно; кнопка **«Очистить сейчас»**.
- `retention.py` (фоновый цикл) + `POST /api/admin/retention/run`.

### ⚡ Масштабирование WebSocket (Redis pub/sub)
- `ws_manager.py` переписан: при нескольких воркерах сообщения и присутствие
  расходятся через Redis (см. ниже). Включается `WEB_CONCURRENCY>1`.

## v2.2 — Каналы-объявления + Ручные статусы

### 📢 Каналы-объявления
Односторонняя рассылка: администратор пишет, остальные участники только читают.
- Тип чата `channel` (поле `Chat.type`).
- Создать: меню (☰) → **«Новый канал»** (доступно только администраторам).
- В канал может писать только **админ канала** (или app-admin). Остальным
  показывается баннер «Только администраторы канала могут публиковать».
- Backend enforce: `POST /api/chats/{id}/messages` отклоняет не-админов в канале
  (403). Флаг `can_post` в `ChatOut` сообщает фронту, показывать ли композер.
- Подписчики получают обычные пуш-уведомления о новых сообщениях канала.

### 🟢 Ручные статусы: «Не беспокоить» / «В отпуске»
- Поле `User.status` (`"" | dnd | vacation`).
- Установить: меню (☰) → **«Мой статус»**.
- `PATCH /api/users/me/status` — сохраняет и рассылает presence-событие всем,
  кто с вами в общих чатах (статус обновляется в реальном времени).
- В шапке приватного чата показывается «🔴 Не беспокоить» / «🌴 В отпуске».

## v2.1 — Пакет улучшений

### 🔔 Уведомления / непрочитанное на уровне ОС
- Desktop-бейдж + подсветка кнопки в панели задач уже поддержаны Electron-оболочкой
  (`setUnread`, `setOverlayIcon`, `flashFrame`) и вызываются из веб-приложения
  (`updateUnreadIndicator`). Браузерные уведомления — через Notification API + SW.

### ✓✓ Индикаторы «доставлено / прочитано»
- `ChatMember.last_read_message_id` отдаётся в `ChatMemberOut`.
- WS-событие `read_receipt` обновляет ✓ → ✓✓ в реальном времени.
- В исходящих сообщениях показывается ✓ (доставлено) / ✓✓ (прочитано всеми).

### ✏️ Черновики сообщений
- Недописанный текст сохраняется по каждому чату в `localStorage` (ключ `cc_drafts`)
  и восстанавливается при открытии чата; очищается после отправки.

### @ Упоминания
- Детект `@username` на сервере (только участники чата), список ID в WS-поле
  `mentions`. Автокомплит при наборе `@`, подсветка в тексте, отдельное
  уведомление «вас упомянули» (звучит даже при выключенных общих уведомлениях).

### 📌 Закреплённые (избранные) чаты
- `ChatMember.is_pinned`; `POST /api/chats/{id}/pin`. Закреплённые чаты —
  наверху списка, с меткой 📌. Управление через контекстное меню чата (ПКМ).

### 📤 Экспорт переписки
- `GET /api/chats/{id}/messages/export?format=txt|html|json`.
  HTML удобно печатать в PDF (Ctrl+P). Доступно из контекстного меню чата.

### 📊 Метрики и health-check
- `GET /api/metrics` — формат Prometheus (`cc_users_total`, `cc_users_online`,
  `cc_messages_total`, `cc_ws_connections`, `cc_up`).
- Добавлен healthcheck для контейнера nginx.

### 💾 Резервное копирование БД
- Сервис `cc_backup` (в `docker-compose.yml`): `pg_dump` в `./backups` каждые
  `BACKUP_INTERVAL` секунд (по умолчанию 24ч), хранит последние `BACKUP_KEEP` (7).
- Восстановление: `./scripts/restore_db.sh backups/<dump>.sql.gz`.

## Изменённые/добавленные файлы

**Backend**
- `app/models.py` — `User.status`, `ChatMember.is_pinned`.
- `app/database.py` — миграции новых колонок.
- `app/schemas.py` — `status`, `is_pinned`, `can_post`, `last_read_message_id`, `StatusUpdate`.

- `js/api.js` — `togglePinChat`, `exportChat` (+ `downloadAuthed`), `setStatus`.

- `docker-compose.yml` — сервис `cc_backup`, healthcheck nginx.
- `scripts/backup_loop.sh`, `scripts/restore_db.sh`.
